# Gamebase — платформа party-games

Документ задаёт продуктовый контекст и технические принципы. Реализация должна согласовываться с этим описанием.

## 1. Vision & Core Value

Платформа для **локальных многопользовательских игр (Party Games)**: телефон каждого игрока — контроллер, общий экран (ТВ/ноутбук) — игровое поле.

**Аналоги:** Jackbox Games, Kahoot, AirConsole, «100 к 1».

**Главная фишка:** нулевой порог входа — без скачиваний. **QR-код → браузер → игра.**

**Рынок:** РФ (локальный юмор, актуальные мемы, интеграция с локальными сервисами — по мере развития продукта).

## 2. User Roles & Experience (UX)

### Host (экран трансляции)

- Запуск на большом экране.
- QR для входа, лобби, анимации баллов, таймеры, финальные результаты.
- **Vibe:** динамично, ярко, «телевизионное шоу».

### Player (мобильный контроллер)

- Минималистичный UI: крупные кнопки, поля ввода, тактильный отклик (вибрация при ответе).
- Вопросы и прогресс — на экране хоста, не перегружать телефон.
- **Vibe:** геймпад в кармане.

## 3. Technical Strategy

### Scalability (ядро vs игры)

- **Ядро:** управление комнатами, сокеты, идентификация сессий.
- **Игры:** подключаемые модули в `/games`; добавление игры не требует переписывания логики комнат.

### Real-time

- Низкая задержка; транспорт: **Socket.io**.

### State management

- **Сервер — source of truth.** Клиент отображает состояние с сервера и шлёт события действий.

### Session persistence

- При закрытии вкладки или обрыве сети игрок возвращается в ту же комнату под своим именем: **localStorage + sessionId** (согласование с сервером).

## 4. Game Mechanics — Phase 1: «Popular Choice» (CommonGuess)

Аналог «100 к 1» / SSGame:

1. Вопрос → игроки вводят ответы с телефонов.
2. Сервер **нормализует** ответы (lower case, trim, fuzzy-слияние опечаток — в логике игры).
3. Группировка похожих ответов → баллы тем, кто попал в **мнение большинства** (самый популярный вариант среди присутствующих).

## 5. UI/UX Principles (Design-First)

- **Mobile first** для интерфейса игрока.
- **Motion:** переходы лобби → игра → результаты — анимации (**Framer Motion** на фронте).
- **Feedback:** игрок явно видит, что ответ принят.

## 6. События сокетов (базовый контракт)

| Событие        | Назначение                          |
|----------------|-------------------------------------|
| `join_room`    | Вход в комнату по коду              |
| `start_game`   | Старт игры (хост)                   |
| `player_action`| Действие игрока в текущей игре      |
| `game_update`  | Снимок/патч состояния с сервера     |

Детали payload расширяются по мере реализации игр; ядро комнаты остаётся стабильным.
